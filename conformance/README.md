# purv conformance harness

Conformance is **signature agreement against a golden model**. RISCOF compiles
each architectural test once, runs it on two plugins — a `DUT` and a
`reference` — and diffs the signature memory each writes. Identical signatures
across the suite = conformant for that ISA string.

## Layout

```
conformance/
  config.bringup.ini   Spike (DUT) vs Sail (reference) — proves the harness
  config.purv.ini      purv  (DUT) vs Sail (reference) — the real run
  sail_cSim/           Sail reference plugin   (generated by scripts/bootstrap.sh)
  spike/               Spike plugin            (generated by riscof setup)
  plugin-purv/         purv DUT plugin
    riscof_purv.py     invocation + signature extraction (1 TODO: purv's CLI)
    purv_isa.yaml      declares the ISA under test (RV32IMC_Zicsr_Zifencei)
    purv_platform.yaml mtime/mtimecmp/reset platform facts
    env/link.ld        test linker script (signature + tohost sections)
    env/model_test.h   RVMODEL_* halt/signature hooks
```

## Why Spike-vs-Sail first

RISCOF has no notion of a "real" DUT vs "fake" reference — both are just
plugins. So before purv exists you run **Spike as DUT against Sail as
reference** (`make validate`). When their signatures match across the whole
suite, you've proven the suite, the linker bracketing, and signature
extraction are all correct. Then swap the Spike DUT for the purv plugin
(`make conformance`) and keep Sail as the reference.

## Selecting featuresets

The ISA string in `purv_isa.yaml` (and `misa`) decides which test buckets run.
RISCOF picks the matching directories from `riscv-arch-test/riscv-test-suite`:

| Extension        | Suite bucket                  |
|------------------|-------------------------------|
| I (base)         | `rv32i_m/I`                   |
| C (compressed)   | `rv32i_m/C`                   |
| M (mul/div)      | `rv32i_m/M`                   |
| Zicsr            | `rv32i_m/privilege`, `Zicsr`  |
| Zifencei         | `rv32i_m/Zifencei`            |

Add an extension to purv → add its letter to `ISA`/`misa` → its bucket runs.

## RV32C watch-list

The C extension is almost entirely a decode/expansion problem, so the bugs the
signature diff will catch cluster here:

- every 16-bit op must expand to the correct 32-bit op (diff against Sail);
- 2-byte instruction alignment becomes legal with C (4-byte without);
- reserved encodings: `c.jr`/`c.jalr` with `rs1=x0`, `c.addi4spn`/`c.lui`
  with immediate `0`, the `RES`/`HINT`/`NSE` cases;
- immediate scaling/sign-extension in `c.addi16sp`, `c.lwsp`, `c.swsp`;
- fetching a 16-bit instruction that straddles a page boundary.

## Bring-up

```
make bootstrap     # fetch submodules + install riscof  (needs open git egress)
make spike sail    # build the two reference emulators
make validate      # Spike-DUT vs Sail-ref
make conformance   # purv-DUT  vs Sail-ref
```
